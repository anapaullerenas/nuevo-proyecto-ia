# Modelo financiero de créditos para la plataforma

## Resumen

La plataforma funcionará con un sistema de créditos. Cada usuaria recibe créditos iniciales para probar la herramienta y, cuando se terminan, puede comprar más saldo por medio de recargas.

La recomendación es iniciar cada cuenta con 600 créditos visibles, pero proteger el gasto real con un tope interno de $3.00 por usuaria. Esto permite que la usuaria sienta que puede usar la plataforma ampliamente, mientras el negocio mantiene control sobre el costo de inteligencia artificial.

El objetivo no es cobrar caro desde el inicio. El objetivo es que la usuaria use la plataforma, vea valor, genere ideas, analice contenido, cree imágenes y después compre créditos para seguir trabajando.

## Cómo funciona el negocio

1. La usuaria entra a la plataforma.
2. Recibe 600 créditos iniciales.
3. Usa esos créditos para chat, guiones, análisis, fichas e imágenes.
4. La plataforma mide internamente cuánto costó cada acción en dólares reales de IA.
5. Si la usuaria llega al tope real incluido de $3.00, se pausa el uso con IA.
6. La usuaria solicita una recarga por WhatsApp.
7. La administradora aprueba el pago desde el panel.
8. Los créditos se abonan a la cuenta y la usuaria sigue usando la plataforma.

Este flujo permite dar una prueba generosa sin dejar abierto el gasto.

## Créditos iniciales

Cada usuaria nueva recibe:

| Concepto | Valor |
| --- | ---: |
| Créditos iniciales visibles | 600 créditos |
| Gasto real máximo incluido | $3.00 USD |
| Saldo comprado inicial | 0 créditos |

Los 600 créditos sirven para que la usuaria pueda probar la plataforma completa. El límite real de $3.00 protege al negocio en caso de que una usuaria use funciones más costosas, especialmente imágenes.

La lógica es: las funciones de pensamiento y análisis deben ser baratas para fomentar uso constante. Las imágenes consumen más créditos porque usan el modelo más avanzado y representan el entregable de mayor valor.

## Margen esperado

La meta recomendada es mantener un margen general de 60% a 70%, favoreciendo el uso de la plataforma.

| Tipo de uso | Ingreso | Costo estimado de IA | Margen aproximado |
| --- | ---: | ---: | ---: |
| Uso balanceado | $10.00 | $3.00 a $4.00 | 60% a 70% |
| Mucho chat y análisis | $10.00 | $1.50 a $3.00 | 70% a 85% |
| Muchas imágenes estándar | $10.00 | $3.00 a $4.50 | 55% a 70% |
| Muchas imágenes altas | $10.00 | $4.00 a $5.50 | 45% a 60% |

La imagen alta puede tener menos margen, pero genera más valor percibido. El margen global se sostiene porque chat, guiones, análisis y fichas usan modelos más baratos.

## Controles financieros

Para que el negocio sea controlado, el sistema debe tener estos candados:

| Candado | Para qué sirve |
| --- | --- |
| Tope real de $3.00 en la prueba | Evita que una usuaria gratuita genere gasto excesivo. |
| Cobro automático de créditos | Cada acción descuenta saldo antes de ejecutarse. |
| Registro de costo real | Administración ve cuánto costó cada usuaria en dólares. |
| Límite diario de uso | Evita abuso o uso accidental excesivo. |
| Límite mensual global de API | Protege el gasto total de la plataforma. |
| Recargas manuales aprobadas | Solo se abona saldo cuando el pago está confirmado. |

## Qué ve la usuaria

La usuaria debe ver:

- créditos disponibles;
- cuánto cuesta cada función;
- historial de consumo;
- paquetes de recarga;
- botón o enlace para contactar por WhatsApp cuando necesite más saldo.

La usuaria no necesita ver el costo real de IA. Ese dato es interno del negocio.

## Qué ve la administradora

La administradora debe ver:

- créditos consumidos por usuaria;
- costo real en dólares por usuaria;
- ingresos por recargas;
- margen aproximado;
- consumo por función;
- usuarios cerca del límite incluido de $3.00;
- usuarios que usan muchas imágenes;
- opción para abonar créditos manualmente.
